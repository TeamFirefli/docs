## Overview

Vercel is ideal for developers who want Git-based workflows, preview deployments, and easy customization. This guide walks you through a one-click deployment.

<Info>
  Vercel deployments are free and scale automatically. You only pay for database hosting (Neon, Supabase, Railway, etc.).
</Info>

## Prerequisites

- A [Vercel](https://vercel.com/signup) account
- A [GitHub](https://github.com/) account
- A PostgreSQL database (Neon, Supabase, Railway, or similar)

## Environment variables

During deployment, Vercel will ask for these required variables:

<CardGroup cols={2} gap="medium">
  <Card title="SESSION_SECRET" icon="key-hole">
    A strong random string (32+ characters). Generate one: <code>openssl rand -base64 32</code>
  </Card>
  <Card title="DATABASE_URL" icon="database-zap">
    PostgreSQL connection string from your database provider (Neon, Supabase, Railway, etc.).
  </Card>
</CardGroup>

<Tip>
  Store these values securely. Never commit them to Git. Vercel's environment variable UI keeps them encrypted.
</Tip>

## After deployment

<Steps>
  <Step>
    <h4>GitHub fork created</h4>
    <p>Vercel will create a GitHub fork of Orbit in your account.</p>
  </Step>
  <Step>
    <h4>Instance is live</h4>
    <p>Your instance is live at `your-project.vercel.app`.</p>
  </Step>
  <Step>
    <h4>Complete setup wizard</h4>
    <p>Go to your instance and complete the setup wizard (Roblox Group ID, username, password).</p>
  </Step>
  <Step>
    <h4>Deploy updates</h4>
    <p>Push code changes to trigger automatic re-deployments.</p>
  </Step>
</Steps>

<Tip>
  Use Vercel's preview deployments to test changes before merging to production.
</Tip>
