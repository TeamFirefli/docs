## Summary

Fetch workspace sessions organised by date. Filter by date range and session category to find specific events.

Session `status` is computed automatically:

| **Status**    | **Description**                        |
| :------------ | :------------------------------------- |
| `scheduled`   | Session is upcoming                    |
| `in-progress` | Session start time has been met        |
| `ended`       | Session has passed and ended           |
| `missed`      | Session passed with no assignments     |

## Response Structure

Each session object includes:

### `type` object
- `id`, `description`, `category`, `gameId`
- **`slots`** — role slots defined on the session type:
  - `id`: Slot identifier (UUID from the role set)
  - `name`: Display name of the role
  - `slots`: Number of available slots
  - `hostRole`: `"primary"`, `"secondary"`, or `null` — indicates host designation
  - `weight`: Display order weight (lower = higher up)
  - `categoryId`, `categoryName`, `categoryWeight`: Category grouping info

### `primaryHost`
The user assigned to the slot with `hostRole: "primary"`. `null` if unassigned.
- `userId`, `username`, `thumbnail`
- `roleId`: The slot ID they are assigned to
- `slot`: Slot index

### `secondaryHosts`
Array of users assigned to slots with `hostRole: "secondary"` (co-hosts).
- `userId`, `username`, `thumbnail`
- `roleId`: The slot ID they are assigned to
- `slot`: Slot index

### `participants`
Array of users assigned to slots with **no host designation** (`hostRole: null`) — regular role assignments only.
- `userId`, `username`, `thumbnail`
- `roleId`: The slot ID
- `roleName`: Display name of the slot
- `slot`: Slot index
- `weight`, `categoryId`, `categoryName`, `categoryWeight`
