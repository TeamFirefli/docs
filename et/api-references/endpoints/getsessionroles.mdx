## Summary

Fetch all role slots defined on a session's type along with their current user assignments. Use this endpoint to:
- Get slot UUIDs needed for [Assign Session Roles](/api-references/endpoints/assignsessionroles)
- Check which slots are occupied and how many remain
- See `hostRole` to identify primary/secondary host slots vs regular participant slots

## Response Structure

The `roles` array contains every slot defined on the session type:

| Field | Type | Description |
| :---- | :--- | :---------- |
| `roleId` | string (UUID) | Slot identifier — use this as `roleId` in assign requests |
| `roleName` | string | Display name of the role |
| `hostRole` | `"primary"` \| `"secondary"` \| `null` | Host designation — `null` for regular participant roles |
| `maxSlots` | integer | Total number of slots available for this role |
| `weight` | integer | Display order weight (lower = higher up) |
| `categoryId` | string \| null | Category UUID, or `null` for uncategorised |
| `categoryName` | string \| null | Category display name |
| `categoryWeight` | integer | Display order weight of the category |
| `assignments` | array | Users currently assigned to this role |

Each entry in `assignments`:
- `userId`, `username`, `displayName`, `picture`
- `slot` — the zero-based slot index the user occupies
