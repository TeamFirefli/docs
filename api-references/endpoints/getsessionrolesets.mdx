## Summary

Fetch all active (non-archived) session role sets defined in a workspace. Role sets are reusable role definitions configured in workspace settings that define what roles are available in a session — including their host designation (primary, secondary, or none), slot count, weight, and associated Roblox group roles.

Use the returned `id` values when creating sessions inline via `roleSetIds` in the [Create Unscheduled Session](/api-references/endpoints/createunscheduledsession) and [Create Scheduled Session](/api-references/endpoints/createscheduledsession) endpoints.

## Response Structure

Each role set object includes:

- **`id`**: UUID — pass this in `roleSetIds` when creating a session
- **`name`**: Display name of the role (e.g. "Co-Host", "Trainer")
- **`hostRole`**: `"primary"`, `"secondary"`, or `null`
  - `"primary"` — this role is the main host of the session
  - `"secondary"` — this role is a co-host / secondary host
  - `null` — regular participant role (not a host role)
- **`slots`**: Number of available slots for this role
- **`weight`**: Display order weight (lower = higher up)
- **`groupRoles`**: Array of Roblox group role IDs that are eligible for this role
- **`category`**: Category this role belongs to, or `null`
  - `id`: Category UUID
  - `name`: Category display name
  - `weight`: Category display order weight

## Typical Workflow

1. Call this endpoint to list available role sets and note the `id` of each role you need
2. Use those IDs in `roleSetIds` when creating a new session type inline
3. The session will be created with role slots that match the workspace's configured role definitions
