### Firefli paigaldamine sinu serveris

<Warning>
  Sul läheb vaja põhiteadmisi terminali kasutuses ja NextJS's.\
  Team Firefli **ei vastuta** seadistamise käigus või kasutamises ilmunud kasutaja vigades.
</Warning>

### Eeltingimused

- Arvuti, millel on võimalik käivitada käsureaülesandeid (nt Ubuntu)
- Root õigused
- PostgreSQL andmebaas
- Docker Compose
- Domeeni ja SSL krüpteerimine

### Firefli seadistamine

Kui oled ühenduses oma serveriga, järgi allolevat käskude juhendit sinu käsurea (Terminal/Console) seansis.

<Steps>
  <Step>
    <h4>Uuenda server</h4>

```
sudo apt update
sudo apt upgrade
```
  </Step>
  
  <Step>
    <h4>Klooni Firefli projekt</h4>

```
git clone https://github.com/TeamFirefli/firefli && cd firefli
```
  </Step>
<Step>
<h4>.env faili seadistamine</h4>
Lihtsalt käivita see käsk oma konsoolis.

```bash
echo "SESSION_SECRET=$(openssl rand -base64 36 | tr -d '\n')" >> .env
echo "POSTGRES_PASSWORD=$(openssl rand -base64 36 | tr -d '\n')" >> .env
echo 'POSTGRES_DB=Firefli' >> .env
echo 'POSTGRES_USER=Firefli' >> .env
echo 'POSTGRES_PORT=5432' >> .env
echo "ENCRYPTION_KEY=$(openssl rand -base64 36 | tr -d '\n')" >> .env
```

Pead ka käivitama selle käsu, asendades 'sinudomeen' URLiga, mida kasutad. Ära unusta lisada protokolli. (http/https)
```bash
echo "NEXTAUTH_URL='sinudomeen'"```
</Step>


<Step>
  <h4>Firefli käivitamine</h4>

  ```
docker compose up --build -d
  ```
</Step>

<Step>
<h4>Andmebaasi käivitamine</h4>

Käivita järgnev käsk:

```bash
docker exec -it firefli-firefli-1 npx prisma migrate deploy
docker exec -it firefli-firefli-1 npx prisma generate
```
</Step>


  <Step>
    <h4>Lõpeta seadistamine veebiliidese kaudu</h4>

    <p>1. Mine https://sinu.domeen.ee - Pead olema SSL</p>
    <p>2. Sisesta oma **Roblox Kogukonna ID**</p>
    <p>3. Sisesta oma **Roblox kasutajanimi** ja parool</p>
    <p>4. 🎉 Kõik valmis!</p>
  </Step>
</Steps>
