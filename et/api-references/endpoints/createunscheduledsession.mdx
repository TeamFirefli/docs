## Summary

Create an unscheduled (one-time) session. You can reference an existing session type by ID or create a new session type inline by referencing role sets from the workspace.

To create inline, first use [Get Session Role Sets](/api-references/endpoints/getsessionrolesets) to retrieve the IDs of the roles you want to include.

## Two Creation Modes

### Mode 1: Use Existing Session Type

Reference an existing session type by ID:

```json
{
  "sessionTypeId": "existing-uuid-here",
  "date": "2026-03-15",
  "time": "14:00",
  "ownerId": "123456"
}
```

### Mode 2: Create Session Type Inline

Define a new session type using role set IDs from the workspace:

```json
{
  "sessionType": {
    "name": "Special Training",
    "description": "One-time special training event",
    "roleSetIds": [
      "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "b2c3d4e5-f6a7-8901-bcde-f12345678901"
    ],
    "statues": [
      {
        "id": "completed",
        "name": "Completed",
        "color": "green",
        "timeAfter": 0
      }
    ]
  },
  "date": "2026-03-15",
  "time": "14:00",
  "ownerId": "123456"
}
```

Use [Get Session Role Sets](/api-references/endpoints/getsessionrolesets) to find the IDs. Each role set carries its name, host role designation (primary/secondary/none), slot count, and category.

## Parameters

### Required
- **date**: Session date in YYYY-MM-DD format
- **sessionTypeId** OR **sessionType**: Must provide one (not both)

### Optional
- **time**: Session time in HH:MM format (24-hour UTC). Defaults to 00:00 if omitted
- **name**: Custom session name (overrides session type name)
- **type**: Session category (default: `"other"`)
- **ownerId**: Session host user ID (Roblox user ID)
- **duration**: Session length in minutes (default: 30)
- **sessionTagId**: Tag UUID for categorisation
- **logs**: Array of initial log entries

## Session Type Object (inline creation)

- **name** *(required)*: Session type name
- **description**: Session type description
- **roleSetIds**: Array of role set UUIDs from [Get Session Role Sets](/api-references/endpoints/getsessionrolesets). Each role set defines the role name, host designation, slot count, and eligible Roblox group roles. Omit or pass an empty array for a session with no role slots.
- **statues**: Array of custom status definitions
  - `id`: Status identifier
  - `name`: Status display name
  - `color`: Status colour
  - `timeAfter`: Minutes after session start when this status activates

## Activity Logging

The endpoint automatically creates:
1. A `session_created` log entry with creation metadata
2. Any additional log entries provided in the `logs` array

## Example: Training with Role Sets

```json
{
  "sessionType": {
    "name": "Advanced Combat Training",
    "roleSetIds": [
      "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "b2c3d4e5-f6a7-8901-bcde-f12345678901",
      "c3d4e5f6-a7b8-9012-cdef-123456789012"
    ]
  },
  "date": "2026-03-20",
  "time": "15:30",
  "ownerId": "123456",
  "duration": 60,
  "logs": [
    {
      "action": "preparation_started",
      "metadata": {"note": "Setting up training area"}
    }
  ]
}
```

This creates a session whose role slots are driven by the workspace role set configuration, including primary/secondary host designations.
